Project Overview

Why MBP detection matters

Magnetic Bright Points are small, high-intensity features in the solar photosphere associated with concentrated magnetic flux. They are linked to solar activity such as solar flares and coronal mass ejections, which can affect satellites and power grids on Earth.

This project explores how deep learning can make MBP detection and classification more scalable, reproducible, and measurable than manual or purely physics-based approaches.

Dataset

Solar image data and labels

The dataset used ROSA G-Band observations from the Dunn Solar Telescope at NMSU across four observation sessions. A tracking algorithm detected MBP candidates, and 1,917 unique tracks were manually labeled as valid or invalid/false positive.

The class distribution is imbalanced, with approximately 90% valid MBPs and 10% invalid MBPs, making careful metric selection and weighted training important.

Dataset snapshot

  • 3,761 total frames
  • 48,713 MBP annotations
  • 500 x 500 px frame size
  • 64 x 64 YOLO-format bounding boxes
  • Valid vs. invalid MBP classification
Models & Results

Patch classification and full-frame detection

The project compares multiple model families. CNN and Tiny ViT perform patch-level binary classification, CNN-LSTM adds temporal modeling, and YOLOv8 performs full-frame object detection.

Baseline CNN

Best patch classifier

Track-based splitting produced stronger results than date-based splitting, with CNN v2 reaching 0.937 accuracy and 0.965 F1 for valid MBPs.

Tiny ViT

Transformer comparison

The CNN outperformed the Tiny ViT while using fewer parameters, suggesting that convolutional inductive bias was more useful for this small domain-specific dataset.

CNN-LSTM

Temporal model

The temporal model used sequence information and produced very strong results, including 0.9968 accuracy and 0.9983 F1 score.

YOLOv8

Full-frame detection

YOLOv8 localized and classified MBPs directly from full frames, providing a complementary capability to patch-level classifiers.

Reflection

What the project shows

The strongest lesson from this work is that model performance depends not only on architecture but also on data splitting, class imbalance handling, and the difference between classification and detection tasks. The project also suggests future work combining YOLO detection with CNN or ViT classification as a two-stage pipeline.
